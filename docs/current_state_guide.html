파일 열기

이 프로젝트를 이해하려면 꼭 알아야 하는 개념

PX4 SITL

실제 드론 FCU 대신 PC 안에서 PX4를 실행하는 방식이다. 비행 제어 로직은 PX4가 담당하고, 우리는 ROS에서 setpoint를 보낸다.

Gazebo Sim Harmonic

현재 Gazebo 계열의 장기 유지 버전이다. Classic보다 구조가 현대적이고, server / GUI / transport 분리가 명확하다.

MAVROS

ROS 2와 PX4를 연결하는 브리지다. state, pose, mode, arming, velocity setpoint를 주고받는다.

OFFBOARD

PX4가 외부 컴퓨터에서 보낸 setpoint를 따라 움직이는 모드다. 현재 autonomy_manager가 이 모드 진입을 담당한다.

LaserScan

2D LiDAR range 배열이다. 현재 perception은 full map을 만들지 않고, 가장 가까운 range만 뽑아 planner에 넘긴다.

Bind Mount

Docker 이미지 안에 코드를 baked 하지 않고, 호스트 저장소를 컨테이너에 그대로 마운트하는 방식이다. 그래서 repo clone이 필수다.

Artifact / Run ID

실행마다 `artifacts/<timestamp>_drone1/` 폴더가 생기고, metadata / summary / csv / event log가 저장된다.

Git Context

metrics_logger는 git commit / branch / dirty 상태를 같이 저장한다. 그래서 “어떤 코드에서 나온 결과인지”를 나중에 연결할 수 있다.

환경 세팅과 재현성

왜 재현성이 중요한가

이 저장소는 단순 실행용이 아니라, 나중에 논문/포트폴리오에서 “어떤 코드 상태에서 어떤 결과가 나왔는지”를 연결할 수 있어야 한다. 그래서 runtime artifact에는 git commit, branch, dirty 상태와 run_id가 같이 저장된다.

  • 코드: git clone 또는 특정 commit checkout
  • 환경: Docker image build 또는 Docker Hub pull
  • 결과: artifacts / experiments ledger

현재 재현 경로

Clone
git clone https://github.com/deepblue21zin/AV_Drone.git
Build or Pull
docker compose build sim ros 또는 Docker Hub pull 후 local tag
Run sim
docker compose up -d --force-recreate sim
Run host GUI
./scripts/run_host_gz_gui.sh
Run ros
docker compose up -d ros -> ros2 launch drone_bringup single_drone_autonomy.launch.py
Validate
./scripts/smoke_test_single_drone.sh

직접 build 방식

git clone https://github.com/deepblue21zin/AV_Drone.git
cd AV_Drone
xhost +local:docker
docker compose build sim
docker compose build ros
docker compose up -d --force-recreate sim
docker compose up -d ros

Docker Hub pull 방식

git clone https://github.com/deepblue21zin/AV_Drone.git
cd AV_Drone
docker pull deepblue2121/av-drone-sim:2026-03-20
docker pull deepblue2121/av-drone-ros:2026-03-20
docker tag deepblue2121/av-drone-sim:2026-03-20 av-drone-sim:latest
docker tag deepblue2121/av-drone-ros:2026-03-20 av-drone-ros:latest
현재 구조는 bind mount 기반이다. 즉 Docker 이미지만 받아서는 충분하지 않고, 반드시 git 저장소도 같이 clone 해야 한다. 이미지에는 “환경”이 들어가고, 실제 프로젝트 코드는 저장소에서 들어간다.

실행 결과는 어디에 남나

Artifacts

artifacts/<timestamp>_drone1/
├─ metadata.json
├─ metrics.csv
├─ summary.json
└─ events.log

`metrics_logger_node.py`가 자동으로 만든다. `summary.json`에는 mode, phase, counts, p99 주기, git commit이 들어간다.

Experiments

experiments/
├─ index.csv
├─ scenario_table.csv
├─ ledger.csv
└─ plots/<run_id>/

smoke test와 후처리 스크립트가 장부와 그래프를 만든다. 취업/논문에서 “문제 -> 수정 -> 결과”를 보여줄 수 있는 기반이다.
