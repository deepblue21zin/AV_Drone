4. Threading Model
3개 스레드의 역할과 생명주기
Main Thread
Flask 서버 시작
HTTP 요청 수신
RosMonitor API 호출
JSON 응답 반환
↻ 요청마다 반복
Spin Thread (daemon)
rclpy.spin_once()
TF 콜백 실행
서비스 클라이언트 콜백 처리
_tf_frames 갱신
timeout 0.1s
↻ active 동안 반복
Hz Thread (daemon)
발행 토픽 목록 조회
각 토픽에 대해:
ros2 topic hz subprocess
4초 타임아웃 후 kill
_hz_cache 갱신
2초 대기
↻ active 동안 반복
Spin Thread와 Hz Thread는 activate() 시 생성, deactivate() 시 self._active = False로 종료. 모두 daemon thread이므로 프로세스 종료 시 자동 정리.
5. API Endpoints
Flask 서버가 제공하는 REST API
Method Endpoint Parameters Response
POST /api/activate {"domain_id": int} {"status":"ok", "domain_id": int}
POST /api/deactivate — {"status":"ok"}
GET /api/status — {"active": bool, "domain_id": int}
GET /api/topics — [{name, type, publishers, subscribers, hz, pub_nodes, sub_nodes}]
GET /api/services — [{name, type, server_nodes, client_nodes}]
GET /api/actions — [{name, type, server_nodes, client_nodes}]
GET /api/tf — {"frames":[{child,parent}], "tree":[tree_nodes]}
GET /api/config — {"port": int, "update_interval": int}
GET /api/nodes — ["node_name", ...]
GET /api/params/list ?node=/node_name ["param_name", ...]
GET /api/params/get ?node=...&param=... {"type":"String", "value":"..."}
POST /api/params/set {"node":..., "param":..., "value":...} {"success": bool, "message":"..."}
6. Frontend Architecture
index.html 대시보드의 구성과 동작 흐름
Header DOMAIN ID 입력 + Activate/Deactivate + Refresh + Status Badge
▼
Topics 이름 / 타입 / Pub-Sub / Hz
Services 이름 / 타입
Actions 이름 / 타입
TF Tree parent-child 트리
▼
Parameters (Full Width) 노드 선택 → 파라미터 목록 → 인라인 편집 (Edit/Set/Cancel)
▼
Expandable Node Detail (▶ 클릭) 각 행 클릭 → Pub/Sub/Server/Client 노드 이름 펼침
자동 갱신: 설정 가능한 주기 (update_interval 파라미터, 기본 5000ms)
수동 갱신: Refresh 버튼 클릭 → manualRefresh() → 즉시 refreshAll() 호출 + 타이머 리셋
병렬 호출: Promise.all([topics, services, actions, tf])
펼침 상태 유지: expanded Set에 이름을 기록하여 갱신 후에도 상태 보존
파라미터: 자동 갱신 없음 — 노드 선택 또는 «↻ Params» 버튼 클릭 시에만 갱신 → /api/params/list + /api/params/get 병렬 조회 → 인라인 편집 → /api/params/set
