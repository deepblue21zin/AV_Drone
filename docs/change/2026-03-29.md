## 원인 분석

1. X11 인증 누락

- 자동 `gzclient`가 `DISPLAY=:0`은 받았지만 `XAUTHORITY`를 못 읽어서 아래 에러가 났다.
  - `Authorization required, but no authorization protocol specified`
  - `Can't open display :0`
  - `could not connect to display :0`
  - `Could not load the Qt platform plugin "xcb"`

2. startup camera auto-zoom

- 인증을 통과한 뒤에도 Gazebo Classic의 기본 `orbit` startup view가 장면 전체를 과하게 잡아, `user_camera`가 너무 멀리(`z≈194`) 잡혔다.
- 이 상태에선 world가 실제로 살아 있어도 화면상 거의 빈 회색 창처럼 보일 수 있다.

## 검증

- 최신 `sim` 로그에서 `Authorization required`, `Can't open display`, `Qt platform plugin` 에러가 사라진 것을 확인했다.
- 최신 `sim` 프로세스에서 `gzserver`와 `gzclient --verbose`가 자동으로 뜨는 것을 확인했다.
- 최신 live camera pose가 아래처럼 고정된 것을 확인했다.

```text
position {
  x: 16
  y: 0
  z: 30
}
orientation {
  x: 0
  y: 0.57286745309829712
  z: 0
  w: 0.819648027420044
}
```

## 현재 기준 사용 방법

```bash
cd /home/deepblue/AV_Drone
xhost +SI:localuser:root
xhost +local:docker
docker compose up -d --force-recreate sim ros
docker compose logs -f sim
```

- `Startup script returned successfully`가 보이면 Gazebo Classic GUI가 자동으로 떠야 정상이다.
- 예전에 띄운 빈/회색 Gazebo 창이 남아 있으면 그 창은 닫고 새로 자동 실행된 창만 본다.
